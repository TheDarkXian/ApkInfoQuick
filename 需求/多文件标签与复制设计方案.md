# 多文件标签与复制设计方案（前端）

更新时间：2026-04-23

## 1. 目标
在现有 APK 信息页上新增“多文件标签工作区”，并提升信息密度，满足以下用户已确认规则：
- 支持多文件拖入（APK + AAB），上限 10。
- 每个文件独立一个标签，标签名为文件名。
- 文件路径显示在“标签页内容区顶部栏”，顶部栏右侧显示 icon。
- 多文件拖入时按顺序自动解析。
- AAB 当前只占位（文案固定为“占位”），不做真实解析。
- 可复制当前标签内容：纯文本 / JSON（不含 icon 二进制）。
- 提供标签批量操作：关闭当前、关闭其他、清空全部。
- 解析失败标签保留，路径保留，显示错误状态。

## 2. 范围
### 本次实现
- 前端交互与状态管理改造。
- 复制能力（当前标签）。
- 多标签管理能力。
- 高密度卡片布局优化。

### 暂不实现
- AAB 真实解析。
- 后端契约变更。

## 3. 交互设计
### 3.1 顶部控制区
- 左侧：拖拽提示 + 选择文件按钮。
- 右侧操作按钮：
  - `复制文本`（当前标签）
  - `复制 JSON`（当前标签）
  - `关闭当前`
  - `关闭其他`
  - `清空`
- 用户反馈：使用 toast（例如超出 10 个、复制成功、复制失败）。
- 交互规则：禁止浏览器原生弹窗（`alert` / `confirm` / `prompt`），统一使用应用内反馈组件。

### 3.2 标签区
- 每个标签显示：`文件名` + `简短状态`。
- 标签本身不显示路径。
- 路径显示在标签页内容区顶部栏左侧。
- 顶部栏右侧显示 icon（无 icon 时显示占位图标或“无图标”标识）。
- 状态枚举：
  - `待解析`
  - `解析中`
  - `成功`
  - `失败`
  - `AAB占位`

### 3.3 内容区（卡片保持，密度提升）
- 保持现有卡片分区：基础信息、版本、权限、签名、ABI、警告、图标。
- 压缩策略：
  - 缩小卡片 padding。
  - 缩小列表行高与字号。
  - 减少卡片间距。
  - Chip 使用 small 尺寸。
  - 图标区改紧凑样式。

## 4. 数据模型（前端）
新增 `FileTab`（建议）
```ts
interface FileTab {
  id: string;                 // 唯一ID（建议 path + timestamp/hash）
  name: string;               // 文件名
  path: string;               // 完整路径（显示在标签内）
  ext: "apk" | "aab" | "other";
  status: "pending" | "parsing" | "success" | "error" | "placeholder";
  envelope: ApkInfoEnvelope | null;
  localError: string | null;
  createdAt: number;
}
```

核心状态：
- `tabs: FileTab[]`
- `activeTabId: string | null`
- `parseQueue: string[]`（按拖入顺序）
- `isQueueRunning: boolean`

## 5. 解析流程
### 5.1 拖入/选择文件
1. 解析文件列表并按输入顺序遍历。
2. 超过 10 个时：保留前 10 个，额外文件忽略，并用 toast 提示。
3. 生成标签：
   - `.apk` -> `pending` 入队。
   - `.aab` -> `placeholder`，状态文案固定“占位”。
   - 其他后缀 -> `error`（可选，或直接忽略并提示）。
4. 若队列空闲，启动顺序解析。

### 5.2 队列执行（仅 APK）
- 串行：取队首 -> `parsing` -> invoke `parse_apk` -> 更新为 `success/error` -> 下一个。
- 失败不阻塞后续任务。
- 当前激活标签默认切换到最新入队或用户最后操作标签。

## 6. 复制设计（当前标签）
### 6.1 复制文本
- 复制字段：
  - 基础信息（name/path/package/channel...）
  - 版本信息
  - 权限列表
  - 签名信息
  - ABI
  - warnings
  - error 信息（若失败）
- 不复制 icon 二进制，仅可复制 `iconUrl` 文本。
- 字段顺序可配置：从模板文件读取并渲染占位符。
- 模板文件建议：`frontend/templates/copy-text.template.txt`
- 占位符格式：`#field#`，例如：
  - `#packname#`
  - `#product_name#`
  - `#app_name#`
  - `#path#`
  - `#version_name#`
  - `#warnings#`

### 6.2 复制 JSON
- 优先复制：
  - 成功/失败时：`{ path, name, status, envelope, localError }`
  - AAB占位：`{ path, name, status: "placeholder", reason: "AAB_NOT_SUPPORTED_YET" }`
- JSON 使用 `JSON.stringify(obj, null, 2)`。

## 7. 失败与边界
- 失败标签必须保留，且 path 必须可见。
- 标签关闭规则：
  - 关闭当前：删除当前标签，激活相邻标签。
  - 关闭其他：仅保留当前标签。
  - 清空：清空全部标签与队列。
- 当关闭正在解析的当前标签：
  - 可选策略A（推荐）：允许后台继续完成并丢弃结果（通过 id 校验）。
- 二次确认（如“清空全部”）也必须使用应用内确认组件，不得使用 `window.confirm`。

## 8. 验收标准
1. 可拖入多个文件并自动按序解析 APK。
2. AAB 生成独立占位标签，显示路径。
3. 标签数量上限 10，超限有提示。
4. 当前标签可复制文本与 JSON。
5. 解析失败标签保留，错误信息可见。
6. UI 比当前版本更紧凑（卡片保留）。
7. 三个标签操作可用：关闭当前/关闭其他/清空。

## 9. 实施清单（开发任务）
1. `App.tsx`：引入 tabs + queue 状态机。
2. `types/apk.ts`：新增标签模型类型（可新建 `types/tab.ts`）。
3. `services/tauri.ts`：复用现有 `parseApk`；增加批量解析调度逻辑在页面层。
4. 新增复制工具函数：`toPlainText(tab)`、`toJson(tab)`，其中 `toPlainText` 从模板文件读取字段顺序。
5. 增加单测：
   - 队列顺序执行
   - 10 个上限
   - AAB 占位
   - 复制文本/JSON输出
   - 关闭当前/关闭其他/清空行为
6. 新增 toast 通知能力（成功/失败/超限）。

## 10. 详细实施计划（执行版）
### Phase A：状态模型与队列
1. 固化 `FileTab/TabStatus` 类型，区分 `pending/parsing/success/error/placeholder`。
2. 建立 `tabs + activeTabId + parseQueue + isParsing` 四元状态。
3. 封装文件接入规则为纯函数（去重、上限、类型分流），避免页面逻辑分散。
4. 串行解析器采用“取队首 -> 解析 -> 回填 -> finally 释放锁”。
5. 标签关闭后回填前做 `id` 存在校验，防止幽灵结果污染。

### Phase B：紧凑工作区 UI
1. 保留卡片结构但压缩密度（padding/间距/行高/chip）。
2. 标签仅显示 `文件名 + 状态`，路径固定放内容区顶部栏。
3. 顶部栏右侧统一放 icon 与标签级动作（导出图标/重试）。
4. AAB 标签固定展示占位说明，失败标签固定展示错误态与路径。

### Phase C：复制与模板
1. 增加“复制文本/复制 JSON”两个按钮，仅作用当前标签。
2. 纯文本由模板渲染，支持占位符替换与顺序可配置。
3. JSON 导出包含当前标签状态与 envelope（AAB 为占位对象）。

### Phase D：规则与回归
1. 增加工作区规则单测（去重/上限/AAB占位/顺序）。
2. 增加复制渲染单测（占位符、注释行、空值兜底）。
3. 增加前端规则测试：禁止 `alert/confirm/prompt`。
4. 执行 `npm run test -- --run` 与 `npm run build` 作为交付门禁。
